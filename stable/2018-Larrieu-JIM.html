1. Jupiter

- de Philippe Manoury
- pour flûte et électronique en temps-réel
- commande Ircam
- créée en 1987 pour 10 ans de l'Ircam
- flûte : Pierre-André Valade
- électronique : Miller Puckette
- dédicace : à la mémoire de Larry Beauregard

Pièce majeure :

-> ouverture du cycle Sonus Ex Machina
-> plus de 35 minutes
-> histoire riche : 30 ans
-> au moins 7 portages (implémentations)

2. Le routing de Jupiter

- 8 modules de synthèse et traitement +Flûte +Spat
- implémentation lourde
- 41 variables, 21 utilisées
- utilisation intensive (plus de 600 occurences)
- écriture au fil des événements (≠ config. type)

Problème :
On ne peut savoir avec précision quels sont les embranchements des modules qui produisent le signal.

Difficulté de représentation

3. Une solution

Positionnement

-> forte compréhension du patch : jupiterVars.csv
-> simuler le patch avec les qlist (sans computation)
-> de la simulation extraire les informations souhaitées
-> obtenir un document hors-temps

Trois programmes réalisés :

1. quels modules produisent du signal ?
(preState)
2. quelles sont les connexions entre modules ?
(modsConnections)
3. quel est le routing ?

Description globale des programmes

- POO pour définir une structure de données
commune à tous les modules
- création d'une instance par module
- scanne les qlsit evt par evt
- assigne les données aux modules respectifs
- application de règles formelles
- écriture de fichiers de données

3e programme :

- charge les preState et modsConections
- construit représentation exhaustive du routing

flute out(reverb)
noise in(osc,reverb) out(spat)
spat in(noise,osc,reverb)
osc out(noise,spat)
reverb in(flute) out(noise,spat)

- application règles formelles

- si un module n'a aucune sortie, alors il est inactif ;
- si un module, hormis la Reverb, est un treat et n'a aucune entrée, alors il est inactif.

- vers une représentation de plus haut niveau
- principe des chaines de routing : de synth vers le dac ou spat
- algorithme récursif
- fichier final routingChains.txt (1.7 k lignes)

4. Utilisation des résultats
en Analyse musicale

- lecture experte
- augmentation de la partition traditionnelle
- conclusion : précision obtenue trop grande ; un réseau de modules de treat à dépasser
